Input Template Customisation

By default, a simple <input/> element is rendered in place of the <ic-datepicker/> element. This can be easily styled by providing classes to the inputClasses option.

This however does not always suffice, as some frontend frameworks require additional markup to function correctly (or you may wish you use a different element altogether). This can be achieved by providing a templateRef to the <ic-datepicker/> element like so;

Angular Material Template Example 
<ic-datepicker [inputTemplate]="materialInputTemplate"></ic-datepicker>

<-- Custom Template -->
<ng-template #materialInputTemplate let-inputValue="getInputValue()">
  <mat-form-field>
    <input matInput [value]="inputValue" placeholder="Material Example">
  </mat-form-field>
</ng-template>

A custom template is defined and assigned a template variable #materialInputTemplate, which is then provided to [inputTemplate]. This template has access to a context method getInputValue(), the result of which is assigned to a local variable let-inputValue. This leaves inputValue available to the template content.

This custom template can contain any HTML, and is not restricted to form fields.
